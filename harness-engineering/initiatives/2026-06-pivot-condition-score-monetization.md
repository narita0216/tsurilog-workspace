# Initiative: 釣れる度（一般＋種別課金）ピボット — 2026-06

## なぜやるか(目的)

記録/AI依存をやめ、**実際にリテンションを生んでいる「釣れる度のチェック」に寄せて黒字化する**（背景・判断は ADR-0011）。
理想状態：無料の一般スコアが磁石として機能し続け、有料(¥150/月)の種別ごとスコアで「チェックの深さ」に課金が乗り、固定費削減と合わせて**月次黒字**になっている。

## ゴール / 完了条件

- [ ] **黒字化**：固定費 vs（削減額＋サブ収益）が月次プラス。
- [ ] **磁石が健全**：釣れる度再設計後もリピート率が落ちていない（無料スコアを壊していない）。
- [ ] **¥150サブの検証**：種別ごと釣れる度サブが初期転換（まず3–5人払う＝「深さに金が出る」証明）。
- [ ] AIアドバイザー有料は非表示/休止、既存課金者は丁寧に移行済み。
- [ ] 今月末(6月末)目標：B1/B2（種別UI＋¥150サブ）を新ビルドでリリース。

### A. 即時・サーバー側（審査非依存・今日から）
- **A1. 固定費 right-size**（黒字の最速レバー）: サーバーインスタンス見直し（資源グラフのヘッドルーム分を削減）、AI API の burn 抑制（無料枠/安いモデル/キャッシュ）。
- **A2. 釣れる度を“一般重み”に再設計**: 秋/潮が動く/水温・気温20度前後/凪<1m 等の種別横断ヒューリスティック。**config駆動**（後でデータ較正を上乗せできる構造）。
- **A3. 種別ごとスコアロジック**: 魚種テーブル（適水温/時期/潮選好…一般生物学ベース）。選んだ瞬間に**体感でスコアが変わる**こと。

## 設計の鉄則（ADR-0011 より・実装で厳守）

1. 無料の磁石を劣化させない（一般スコアも単体で役立つ）。
2. 記録と基本の釣れる度は永久に無料。
3. 売るのは深さ・便利さ（種別/多日/プッシュ）。
4. 見せ方は robust（方向性・質的）。精密な符号付き分解は出さない。
5. 種別ごとは“体感で違う”こと（relabel にしない）。

## 要件定義フェーズで詰める論点（実装前に確定）

1. **A1**: 現行サーバーの適正インスタンス（どこまで落とせるか）。AI API 実コスト把握。
2. **A2**: 一般重みの初期値（条件ごとヒューリスティック）。config/テーブルのスキーマ。
3. **A3/B1**: 対象魚種の初期セット（地域主要種＝チヌ/メジナ/アジ等）。種別パラメータの出典。
4. **B2**: サブ商品設計（¥150単品 or “チェック深掘り”バンドル＝種別＋多日予報＋プッシュ）。既存課金者の移行手順。
5. **B4**: 記録の最小UX（ワンタップ/ウィジェット/オフライン）。
6. **見せ方**: 一般・種別スコアを robust に出す形式（指標/グラフ/質的ラベル）。

## スコープ外・先送り
- データ較正された釣れる度（記録が貯まってから）。
- 接地AI＝上位プラン（記録/AIが使われる状態になってから最高品質で再投入）。
- 地形/水深マップ（高精度水深データ未調達）、釣りレベル（優先度低）。
