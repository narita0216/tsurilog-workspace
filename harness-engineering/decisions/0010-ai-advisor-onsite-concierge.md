# ADR-0010: AIアドバイザー「現地」をコンシェルジュ化（観察ウィザード＋総合評価＋効果測定）

- 状態: 採用（2026-06-08 ユーザー確定）
- 関連: ADR-0006(AI戦略実行)、釣れる度(GetAnalysisRateController)、ADR-0009(釣り知識)

## 背景・狙い
初心者〜中級者ターゲットで、有料AIアドバイザーの「現地アドバイス」を“課金してでも使いたい”水準に引き上げる。素のLLM/環境APIが持てない**現場の観察情報**（水深・底質・障害物・潮目・離岸流・鳥・周囲の釣れ具合）を、**初心者でも見て答えられる選択式ウィザード**で取得し、環境データ×釣りログ×地点mdと突合して**プロ級の判断**を返す。「質問は初心者・判断はプロ」。

## 入力（現地ウィザード）
- **場所別に分岐**（Q1: 堤防/砂浜/岩場/河口）。各場所で“見て選ぶだけ”の観察質問（◯選択式、必ず「わからない」）。専門用語は使わず内部enumへマップ（言い換え辞書）。
- 共通の締め：足もとの深さ（測り方ガイド＝オモリ落下→たるみ取り→水面に出るまで巻く→巻き回数→目安m）、潮目（色/泡ゴミの線/さざ波境目）、鳥（突っ込み=鳥山）、周囲の釣れ具合＋釣り方。
- 既知（潮/天気/風/波/水温/位置）は**聞かない**（自動取得）。
- **魚種は聞かない**（出力で予測）。
- **現地動画は残す**（任意）。役割: ①ユーザー記録 ②回答の答え合わせ。動画は重いので**Gemini解析結果(video_summary)のみ保持**。
- **自由質問（任意・末尾）**: 既存 free_text と同仕様。

### 観察→好/悪/安全 と プロ信号（例）
- 砂浜「沖へ伸びる白泡の筋」→ ⚠️離岸流（安全=中に入らない／好=魚の通り道）。「くぼみ」→◎かけあがり。
- 岩場「白い泡(サラシ)」→◎サラシ。河口「ヨレ」→◎反転流。堤防「先端/角＋外向き」→◎潮通し、牡蠣殻/テトラ→◯隠れ家＆根がかり注意。
- 共通: 潮目◎/鳥山◎/まづめ◎/濁り○/強風✕/ベタ凪△。

## 出力（上→下） ※スコアはルール計算、文章/魚予測はAI
1. **総合評価（MAX100・アニメ）**
   - `base` = 釣りログだけの「釣れる度」(偏差スコア0-100, 無料機能と同じ)。
   - `AI現地評価 = clamp( base + clamp(Σ好条件加点 − Σ悪条件減点, -30, +30), 0, 100 )`（**現地補正は±30上限**）。
   - 表示: base / AI現地評価 / **差分(±N ↑↓)**。base→評価へゲージが伸びるアニメ。
   - 重みは全て定数で調整可（釣れる度のK/G同様）。安全性は減点に混ぜず別枠（重大危険時のみ頭打ち＋警告）。
2. **現地サマリー**（3カテゴリ・**動的に検出信号を列挙**）
   - 😄 好条件（◎/○ ＋用語＋一言。タップで用語解説＝初心者の学習）
   - 😭 悪条件（△/✕）
   - 🦺 安全性（⚠️。離岸流/高波/滑り等。釣れやすさスコアとは独立して警告）
3. **釣れそうな魚**（魚種 × 可能性★ × サブ「なぜ」「どう釣る(短い初心者語)」）。地点md×現場×釣りログで予測。データ薄は「データ少なめ」表示＋一般化フォールバック。

## 同セッション追加質問
- 一度出力後、**同じ観察＋動画解析結果を裏で保持**し、**自由質問だけ変更して再質問**できる（チャット継続体験）。既存 `ai_strategy_messages`(role/content) を会話ログに活用。動画は再送せず保持済み video_summary を再利用。

## 効果測定（価値証明ループ）
- 出力後、その釣りで**「釣れた？」を1タップ収集**（釣れた/釣れない/まだ など）→ `ai_strategies` に紐付け保存。
- **管理ダッシュボードに表示**（AIアドバイザー利用→釣果のフィードバック率/釣れた率、使った釣行 vs 使わない釣行 の比較指標）。

## 既存資産への接続（壊さない）
- backend: `StrategyService::generateOnSite`/`buildContext`(env/DataRate/Depth/NearestSpot/point md/video/linked/agent)、`ConditionScoreService`(条件factors)、`AnthropicClient`(plan→tier)、`GeminiClient::analyzeVideo`、`AiUsageService`(plan/回数)。
  - 観察(observation)を `OnSiteStrategyRequest` に追加 → buildContext に「現場の観察」ブロック注入。
  - 総合評価は `ConditionScoreService` を拡張 or 新サービスで base(釣れる度)＋好悪重み±30 を算出。
  - 出力JSONに `local_summary{good[],bad[],safety[]}`・`overall{base,onsite,delta}`・`expected_fishes[{name,stars,why,how}]` を追加（既存 StrategyResult は維持しつつ拡張）。
  - 観察・video_summary・feedbackは `ai_strategies` に列追加（または observation JSON 列）。
- native: 既存 `Wizard` を場所分岐の観察ウィザードに拡張、`on-site.tsx` を差し替え。`StrategyResultView` を 総合評価/現地サマリー(好/悪/安全)/釣れそうな魚 に再構成。動画(expo-image-picker)維持。

## 実装ステージ
1. **効果測定ループ**（「釣れた？」: 列追加＋エンドポイント＋native UI＋管理画面表示）← 自己完結・最小・高価値。**先行実装**。
2. **観察スキーマ＋総合評価**（場所別enum/言い換え辞書、好悪安全マッピング＋重み、base±30の計算）。
3. **出力再構成**（local_summary/overall/expected_fishes をプロンプト契約＋StrategyResultView＋総合評価アニメ）。
4. **同セッション追加質問**（observation/video_summary保持→free_textのみ変更で再生成、会話ログ）。

各ステージで backend phpunit / native tsc・lint、API変更時は /contract-check。
