# ADR-0011: 記録/AI依存から「釣れる度（一般＋種別課金）」中心へピボット

- **状態:** 採用（2026-06-20 ユーザー確定）
- **Date:** 2026-06-20
- **Deciders:** プロダクトオーナー（narita）
- **Tags:** strategy, monetization, condition-score, pivot

## Context

- ローンチ後の実データ（リテンション分析 Phase 1）で、**記録（釣行記録の投稿）はほぼ死んでいる**（記録者≒2人）。「記録が貯まる→データ較正→精度向上」のフライホイールは現状回らない。
- 一方で**実際に効いている行動は「釣れる度のチェック」**（2日以上リピート ~47%、活動 ~82%）。リテンションエンジンはここ。
- 直近で出した**有料AIアドバイザー（ADR-0006/0010・釣りログ Plus ¥500/月）は、期待したほど使われず価値が出なかった**。
- 黒字化が急務。AI API/サーバーの固定費が burn している。
- 「データ較正された釣れる度」は記録が貯まるまで作れないが、**公開された釣りの科学＋生物学に基づく“一般論ベースの釣れる度”は今日作れる**（防御は弱いが先行優位は取れる。moat 化は“データ較正された日”に初めて起きる）。

## Options

### Option A: 記録フライホイール＋AIに賭け続ける（現状維持）
- **Pros:** 既存実装を活かせる。長期 moat（データ堀）狙い。
- **Cons:** 記録が増える兆しが無く、フライホイールが回らない。AIは使われず固定費だけ burn。黒字化が遠い。

### Option B: 釣れる度（一般＋種別課金）中心へピボット【採用】
- 無料＝種別非依存の一般スコア（磁石）／有料＝種別ごとスコア（差別化・¥150/月）。
- 釣れる度を**一般重み(config駆動)で再設計**（後でデータ較正を上乗せ可能な構造）。
- AIアドバイザー有料は**一旦廃止**（長期R&Dに格下げ。記録/AIが“使われる状態”になってから最高品質で再投入）。
- 黒字化＝**固定費 right-size（即・確実）＋ 軽いサブスク(¥150)の検証**の合わせ技。
- **Pros:** 実際に効いている行動で monetize。今日作れる・速く出せる。固定費削減は即効。磁石（無料スコア＋記録は永久無料）を壊さない。
- **Cons:** 一般スコアはコモディティ寄り（先行優位どまり）。種別が“体感で違う”を作れないと課金が死ぬ。

### Option C: 課金をやめて広告のみ
- **Pros:** 摩擦ゼロ。
- **Cons:** eCPM 低く黒字化に遠い。「チェックの深さに金が出る」仮説を検証できない。

## Decision

**Option B を採用。** 記録/AI依存をやめ、リテンションを生んでいる「釣れる度のチェック」に寄せて monetize する。無料＝一般スコア（磁石・劣化させない）、有料＝種別ごとスコア（¥150/月・体感で違うこと）。釣れる度は一般重み(config駆動)で再設計し、データ較正は後付け可能にする。AIアドバイザー有料は一旦廃止し長期R&Dへ。黒字化は固定費削減＋軽いサブスク検証で進める。

## Consequences

- **得るもの:** 実際に効いている行動での収益化。今月リリース可能な速度。固定費削減による即時の黒字寄与。防御可能な“今日作れる”価値。
- **失うもの:** AIアドバイザー（有料）の現行価値提案。記録フライホイールへの短期投資。
- **新たに発生する作業:** 釣れる度の一般重み再設計（config/魚種テーブル）、種別選択UI＋種別スコア表示、¥150サブの新設と既存課金者(釣りログ Plus ¥500/AI)の移行、固定費 right-size、リテンション計測の本格運用。詳細 → `initiatives/2026-06-pivot-condition-score-monetization.md`。
- **後戻り可能性:** reversible（AIコードは削除せず非表示/休止。記録は“流通リードをデータ堀に変える分岐スイッチ”として別途実験で残す）。

## 設計の鉄則（実装で絶対に外さない）
1. **無料の磁石を劣化させない。** 種別非依存スコアも単体で役立つ“一般コンディション”にする（わざと薄くして有料へ誘導しない）。
2. **記録（データ投稿）と基本の釣れる度は永久に無料。** フライホイールは有料化しない。
3. 売るのは**深さ・便利さ**（種別／多日／プッシュ）であってコア価値ではない。
4. **釣れる度の見せ方は robust（方向性・質的）。** 精密な符号付き分解は出さない（“精度低い”の急所を自作しない）。
5. **種別ごとは“体感で違う”こと**（relabel にしない）＝課金の生死。
